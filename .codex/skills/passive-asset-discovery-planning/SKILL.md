# Project planning

Read the root README and inspect the current worktree. Define one vertical slice at a time.

For each slice, state the user outcome, in-scope and out-of-scope work, acceptance criteria, affected components, dependencies, risks, and focused validation. Sequence work from deterministic offline PCAP processing to state/output, live capture, then delivery artifacts. Do not schedule Docker, demo, or documentation work as complete before their artifacts are executable and verified.

Keep a slice independently buildable and reviewable. Split cross-cutting work when its prerequisites can be tested separately.
